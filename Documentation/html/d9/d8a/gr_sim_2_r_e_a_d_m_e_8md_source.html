14 System Requirements
15 -----------------------
16 
17 grSim will likely run on a modern dual core PC with a decent graphics card. Typical configuration is:
18 
19 - Dual Core CPU (2.0 Ghz+)
20 - 1GB of RAM
21 - 256MB nVidia or ATI graphics card
22 
23 Note that it may run on lower end equipment though good performance is not guaranteed.

26 Software Requirements
27 ---------------------
28 
29 grSim compiles on Linux (tested on Ubuntu variants only) and Mac OS. It depends on the following libraries:
30 
31 - [CMake](https://cmake.org/) version 2.8+
32 - [OpenGL](https://www.opengl.org)
33 - [Qt4 Development Libraries](https://www.qt.io) version 4.8+
34 - [Open Dynamics Engine (ODE)](http://www.ode.org)
35 - [VarTypes Library](https://github.com/szi/vartypes)
36 - [Google Protobuf](https://github.com/google/protobuf)
37 - [Boost development libraries](http://www.boost.org/) (needed by VarTypes)
38 
39 Please consult the [install instructions](INSTALL.md) for more details.

41 Usage
42 -----
43 
44 Receiving data from grSim is similar to receiving data from [SSL-Vision](https://github.com/RoboCup-SSL/ssl-vision) using [Google Protobuf](https://github.com/google/protobuf) library.
45 Sending data to the simulator is also possible using Google Protobuf. Sample clients are included in [clients](./clients) folder. There are two clients available, *qt-based* and *Java-based*. The native client is compiled during the grSim compilation. To compile the Java client, please consult the corresponding `README` file.
